HOW THIS MAPS TO SCHOOL ROLES
Subject Teacher
    • features/results/ScoreEntry.tsx
    • Can only access:
        ◦ Assigned classes
        ◦ Assigned subjects
Form Teacher
    • features/results/ResultSummary.tsx
    • features/reports/ClassReport.tsx
    • features/attendance/*
    • Add remarks

Admin
    • Manage users, classes, subjects, teachers
    • ResultApproval.tsx
    • Attendance overview
    • Lock results

ROUTING STRUCTURE (IMPORTANT)
Example route grouping:
/login

/dashboard
/dashboard/students
/dashboard/classes
/dashboard/results
/dashboard/results/approve
/dashboard/reports
/dashboard/attendance
With:
    • ProtectedRoute → authenticated users
    • RoleGuard → role-based access

WHY THIS STRUCTURE IS IDEAL
✔ Feature-based (easy to scale)
✔ Clean separation of concerns
✔ Works perfectly with FastAPI
✔ Easy to onboard junior devs/students
✔ Suitable for real school deployment
